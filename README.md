### Project Overview

This semester project was developed as a comprehensive web application that implements the following technologies and concepts:

**Frontend Development:**
- **HTML5 & CSS3**: Structure and styling of the user interface
- **JavaScript (ES6+)**: Dynamic interactivity and asynchronous data processing
- **Single Page Application (SPA)**: Page transitions without page reload
- **Responsive Design**: Adaptation to different screen sizes

**Backend Development:**
- **Node.js**: Server-side JavaScript runtime environment
- **Express.js**: Web framework for API endpoints and routing
- **RESTful API**: Standardized interface for data operations
- **Middleware**: Request processing and error handling

**Database Integration:**
- **MongoDB**: NoSQL database for flexible data storage
- **Mongoose**: ODM (Object Data Modeling) for MongoDB
- **CRUD Operations**: Complete data management (Create, Read, Update, Delete)

### Core Features

1. **User Authentication and Role Management:**
   - Login system with user roles (Admin/Regular User)
   - Role-based access control (RBAC)
   - Secure password management

2. **Location Management:**
   - CRUD operations for environmental locations
   - Geodata integration (addresses, coordinates)
   - Categorization of locations

3. **Real-time Data Updates:**
   - Asynchronous API calls with Fetch API
   - Dynamic UI updates without page reload
   - Error handling and user feedback

4. **Interactive Map:**
   - Integration of Mapbox GL JS for interactive map visualization
   - 2/3 map, 1/3 list layout for optimal overview
   - Hover effects: Mouse over list item highlights corresponding marker
   - Click on location opens detail view
   - Automatic marker updates on address changes
   - Address validation with Nominatim (OpenStreetMap)
   - User-friendly error messages for invalid addresses

5. **Image Management:**
   - Image upload during location creation (optional)
   - Image validation (JPG, PNG, GIF, max. 2MB)
   - Image preview in detail view
   - Adding, replacing, and deleting images
   - Automatic image removal when deleting locations
   - Storage as Base64 in MongoDB
